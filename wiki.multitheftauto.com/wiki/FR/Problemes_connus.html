Client

Général

J'ai la version Steam de GTA Sa. Comment jouer à MTA:SA ?

La version Steam est actuellement incompatible avec MTA:SA. Cependant, avec quelques manipulations ont peut arriver à faire fonctionner MTA:SA.

  • Trouvez un crack GTA SA 1.00 No CD (Google est votre ami). Vous allez obtenir la version 1.00 Cracked/NoCd EXE, ce n'est pas une image du disque. Toute les version HOODLUM fonctionnerons.
  • Ouvrez avec winrar ou d'autres logiciels semblables, à l'intérieur il doit y avoir un gta_sa.exe. Placez ce fichier dans votre dossier d'installation du jeu. Normalement C:\Program Files\Steam\steamapps\common\grand theft auto san andreas. Aucun fichier ne doit être remplacer durant cette manipulation.

Cette action ne vous empêchera pas de jouer via steam et vous permettra de jouer à MTA:SA

Est-ce que MTA:SA fonctionne avec les versions v1.01 or v2.00 de GTA SA ?

Non. Mais il est possible de le faire fonctionner - Rendez-vous sur Topic pour les instructions et patchs nécessaires.

Ecran Noir/Affichage Suspendu au démarrage

  • Suspension de l'affichage au démarrage.

Il est possible que après le logo Grand Theft Auto vous deviez répondre à des boîtes de dialogue pour poursuivre le démarrage, celle ci peuvent être gelées. Tentez de cliquer avec le bouton gauche de votre souris plusieurs fois, ou essayer les touches de votre claviers.

  • Ecran noir après l'écran avec le logo (Possibilité qu'il y ai du texte dans le coin de la fenêtre).

Il est possible que ce soit à cause de votre version de direct X non supportée ou un autre problème au niveau de votre carte graphique. Par défaut le menu dynamique de MTA est activé, tenter de désactiver celui-ci de la manière suivante : ouvrez coreconfig.xml qui se trouve dans GTA San Andreas\MTA, et changez la valeur de menu_options en 248.

Arrêt après l'écran de démarrage

  • Rien ne se passe après le 'Stop playing with yourself'

Si vous utilisez nVidia Geforce, tentez de désactiver "nView Desktop Manager" avant de démarrer MTA.

Tentez aussi de supprimer le fichier ("gta_sa.set") dans "Documents\GTA San Andreas User Files".

Si le problème persiste, vérifiez que votre logiciel anti-virus autorise "multi theft auto.exe" de fonctionner. D'autre logiciel de sécurité peuvent être la source du blocage.

Crash de MTA après l'écran avec le logo

  • MTA crash après 'Stop playing with yourself'. Avant le mode solo et multijoueur fonctionnait !

Téléchargez le dernier DirctX sur Microsoft. Vérifiez aussi dans le gestionnaire de tache (CTRL+ALT+DEL ou Suppr), si le processus gta_sa.exe ne tourne pas déjà. Si oui arrêtez le.

Si vous utilisez une résolution peu commune (Exemple : 960x720), essayez de changer celle-ci (Exemple : 640×480, 800×600, 1024×768, 1152×864, 1280×1024) en lançant Grand Theft Auto San Andreas en mode solo, changez la résolution et quittez.

Si vous utilisez Windows Vista ou Windows 7, essayez ceci :

  • Activer la comptabilité Windows XP SP3 pour Multi Theft Auto.exe et gta_sa.exe, exécutez ces processus en tant qu'administrateur "Run this program as an administrator".
  • Configurez "Data Execution Prevention" : utilisez l'option Turn DEP for all programs and services except those I select. Cliquez sur add et trouvez "Multi Theft Auto.exe" et "gta_sa.exe" pour les ajouter.
  • Lancez les toujours en tant qu'administrateur.

"Fichiers importants modifiés"

  • Vous recevez ce message d'erreur "San Andreas data files have been modified"

Vous pouvez corriger cela en Installant ce patch. Si ca ne fonctionne pas, ré-installer GTA San Andreas.

"Assertion in CMainMenu.cpp line 106" au démarrage de MTASA

  • J’obtiens le message "assertion in CMainMenu.cpp (line 106)" après avoir lancer MTASA. Je peux l'ignorer, mais ensuite le jeu crash quand je rejoins un serveur.

Cela arrive quand GTASA ou MTA:SA est installer dans un répertoire qui contient des caractères ne prenant pas en charge le code ASCII (Exemples : Cyrillic, Polish, Japanese).

Pour résoudre ce problème, ré-installer MTA:SA et GTASA dans un dossier adéquat.

MTA crash si je rejoins un serveur

  • MTASA crash quand je me connecte à un serveur. Mais le mode solo fonctionne !

Les modes installés pour le mode solo peuvent empêcher MTA:SA de focntionner, en créant des crashs - Utilisez toujours une version sans mode pour jouer à MTA:SA.

Cela peut aussi se produire si votre exécutable du jeu proviens de version non supportées (Exemple : 1.0 Allemande ou Australienne). Pour résoudre ce problème, utilisez notre convertisseur.

Mes touches ne fonctionnes pas

  • Impossible d'utiliser les touches de mon clavier.

Essayez d’utiliser la commande 'copygtacontrols' dans la console.

Skin blanc

  • Vos skin sont blancs !

Cette erreur est causée par les statistiques de votre skin. Pour résoudre ce soucis, vérifiez d'avoir mis une valeur différente de 0 à "fat" et à "muscles" quand vous modifiez vos skins.

Mouvements des véhicules incorrects

  • Je ne sais pas conduire comme d'habitude

Soit un script modifie la façon de conduite ou alors le mode de jeu du serveur sur lequel vous jouer est configuré de cette manière.

Paramètres non sauvegardés

  • Mon MTA ne veut pas sauver les paramètres et il crash.

Premièrement, configuré votre MTA comme vous le souhaitez, ensuite quittez le jeu et relancez le. Normalement les paramètres devraient être sauvés automatiquement. Supprimez le fichier "coreconfig.xml" ensuite configurez votre MTA et quittez le jeu.

Joystick / Manette

  • MTA ne reconnait pas mon contrôleur de jeu

Vérifiez que votre contrôleur est configuré comme étant le premier reconnu par windows pour les jeux (MTa n'utilisera que le premier contrôleur). Configurer celui-ci dans les options "Settings" de MTA.

Souris : La vue libre ne fonctionne pas

  • MTA ne reconnait pas ma souris

Plusieurs joueurs rencontre ce problème avec leur souris. Ils peuvent l'utiliser dans les menus, se connecter à un serveur, mais ne peuvent pas l'utiliser en vue libre. Ce problème peut être résolu en cliquant sur la touche retour Windows, actionner le bouton de votre souris puis revenez dans le jeu. Si cela ne fonctionne pas tentez de démarrer GTA en mode solo, allez dans les options dans "controler setup" et mettez "Mouse + Keys" à la place de "Joypad".

La liste des serveurs ne fonctionne pas

  • la liste de "Browse Servers" est en "Loading" mais les serveurs n'apparaissent pas

Cela peut dépendre de votre connexion, il faut parfois quelques secondes pour que celle ci apparaisse entière. Veuillez patienter un peu.

Si elle n'apparaît pas, logué vous sur GameMonitor, et cliquez sur le bouton vert "Play" pour rejoindre un serveur.

Serial invalide

  • Je reçois le message d'erreur 'Invalid serial number' quand je lance MTA

Vous utilisez une version dépassée de Multi Theft Auto. Téléchargez la plus récente sur MTA:SA

'Network module not compatible!' au démarrage de MTA:SA

  • J'ai le message d'erreur 'Network module not compatible!' quand je lance MTA:SA

Cela veut dire que MTA:SA est corrompu. Ré-installez le.

Message 'No such mod installed (deathmatch)'

  • J'ai le message d'erreur 'No such mod installed (deathmatch)' quand je me connecte à un serveur

Solution 1 : Ré-installez MTA.
Solution 2 : Exécutez MTA:SA en tant qu'administrateur.

D3dx9_**.dll

  • Quand je lance Multi Theft Auto: San Andreas j'ai un message d'erreur m'indiquant que D3dx9_**.dll (** = un nombre) est introuvable.

Cela signifie que DirectX n'est pas installer ou est dépassé. Pour l'installer ou le mettre à jour rendez-vous sur DirectX.

Erreur CRC mismatch

  • Quand je rejoins un serveur, la chatbox envoi 'CRC mismatch'

Ceci est un problème au niveau du serveur. Contactez le propriétaire du serveur.

Message 'Error loading network module!' au démarrage de MTA

  • Message d'erreur :

Error loading network module! (The application has failed to start because its side-by-side configuration is incorrect. Please see the application event log for more detail.

Vérifiez que Microsoft Visual C++ 2008 SP1 redistributable packages (x86) soit bien installé sur votre machine :

Microsoft Visual C++ 2008 Redistributable package (x86)

Microsoft Visual C++ 2008 SP1 Redistributable package (x86)

Windows Vista® / Windows 7- Spécifications

Crash lors de la connexion

  • Je crash quand je rejoins un serveur - sous Vista

Il semble que le soucis vienne de "Microsoft DirectX April 2006 SDK Redistributable DLL file" (d3dx9_30.dll). Vérifiez que la mode de comptabilité de vos deux exécutables (celui du jeu et celui de MTA) soient bien désactivés.

Message : Clock manipulation error

  • Je reçois le message d'erreur 'Clock manipulation detected!' quand je lance MTA:SA

Cela est causé par une mauvaise configuration de votre système au niveau de la date. Vérifiez vos paramètres de date et heures et le problème devrait être réglé.

Cela peut aussi arriver si vous utiliser le processeur AMD Athlon 64 X2 avec de vieux drivers. Mettez les à jour via AMD's Website.

Arrêt au lancement

  • Quand je démarre MTA:SA, rien ne se passe (GTA_SA.exe tourne mais ne se lance pas)

Lancez MTA:SA en tant qu'administrateur. Pour celà, clic gauche sur l'éxécutable, allez dans 'Propriétés', puis dans l'onglet 'Mode de Compatibilité' cochez la case.

Problèmes Généraux de GTA

  • J'ai des crashs et soucis inexplicables

Vérifiez que votre ordinateur à les caractéristiques nécessaires pour faire tourner GTA:SA sur Caractéristiques requises. GTA:SA fonctionne parfaitement avec les modes de comptabilités de Windows 98/2000/XP/2003.
